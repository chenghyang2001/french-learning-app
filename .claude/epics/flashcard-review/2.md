# Task: 新增 PATCH /api/words/:id/review 端點（SM-2 更新邏輯）

## Description

在現有 `api/words.js` 中新增 `PATCH /:id/review` 路由，接收複習結果（`known: true/false`），依最小化 SM-2 演算法更新 `words.srs_interval` 與 `words.next_review_at`，並回傳更新後的欄位。

**不需要新檔案、不需要 migration**——`words` 表已有 `srs_interval` 與 `next_review_at` 欄位（見 `db/schema.sql`）。

## Acceptance Criteria

- [ ] `PATCH /api/words/:id/review` 接受 body `{ "known": true }` 或 `{ "known": false }`
- [ ] `known: true` → `srs_interval = min(srs_interval * 2, 30)`，`next_review_at = date('now', '+N days')`
- [ ] `known: false` → `srs_interval = 1`，`next_review_at = date('now', '+1 days')`
- [ ] 回傳 `{ id, srs_interval, next_review_at }` 狀態 200
- [ ] `id` 不存在 → 404 `{ error: "找不到單字" }`
- [ ] `known` 不是 boolean → 400 `{ error: "known 必須是 boolean" }`

## Technical Details

```javascript
// 新增至 api/words.js 現有路由末端

router.patch("/:id/review", (req, res) => {
  const id = Number(req.params.id);
  const { known } = req.body;

  // 驗證
  if (!Number.isInteger(id) || id <= 0) return res.status(400).json({ error: "無效的 id" });
  if (typeof known !== "boolean") return res.status(400).json({ error: "known 必須是 boolean" });

  const db = getDb();
  const word = db.prepare("SELECT srs_interval FROM words WHERE id = ?").get(id);
  if (!word) return res.status(404).json({ error: "找不到單字" });

  const newInterval = known ? Math.min(word.srs_interval * 2, 30) : 1;
  db.prepare(
    "UPDATE words SET srs_interval = ?, next_review_at = date('now', '+' || ? || ' days') WHERE id = ?"
  ).run(newInterval, newInterval, id);

  const updated = db.prepare("SELECT id, srs_interval, next_review_at FROM words WHERE id = ?").get(id);
  res.json(updated);
});
```

**注意**：`api/words.js` 用 CommonJS（`require`）+ Express Router。`getDb()` 已在此檔頂部 import。
