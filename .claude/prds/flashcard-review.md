# PRD: flashcard-review

## Executive Summary

為法語學習 App 加入閃卡複習功能。使用者每次複習一批單字閃卡，正面顯示法文、背面顯示中文，按「認識 / 不認識」記錄學習狀態，系統依答題結果決定下次複習時間（間隔重複）。目標是提高「複習 20 張後已掌握比例」，解決單字學了很快就忘的核心痛點。

## Problem Statement

目前 App 有測驗功能，但缺少主動「複習」機制：

- 使用者學過的單字沒有追蹤記憶狀態
- 不知道哪些單字已熟悉、哪些需要重複練習
- 學完即忘，缺乏間隔重複的強化機制

## User Stories

1. **複習閃卡**
   身為學習者，我想每次看到一張閃卡（正面法文），翻面確認中文後標記「認識 / 不認識」，讓系統記住我的學習狀態。
   - AC1: 閃卡正面顯示法文單字（含詞性）
   - AC2: 點擊「翻面」後顯示中文翻譯
   - AC3: 可按「認識」或「不認識」記錄結果
   - AC4: 記錄結果後自動跳到下一張

2. **看到每日複習佇列**
   身為學習者，我想打開 App 就看到「今天需要複習 N 張單字」，快速進入複習。
   - AC1: 首頁顯示今日待複習數量
   - AC2: 點擊進入閃卡複習頁面
   - AC3: 若無待複習單字，顯示「今天的複習已完成！」

3. **完成後看到掌握比例**
   身為學習者，複習完一輪後想看到「已掌握 X / 20」的摘要，確認進步。
   - AC1: 複習完畢後顯示本輪統計（認識 N 張 / 不認識 M 張）
   - AC2: 顯示累計掌握率（歷史資料）

## Functional Requirements

1. **單字資料來源**：讀取現有 SQLite DB 中的法語單字（與 `api/words.js` 共用）
2. **學習狀態追蹤**：新增 `flashcard_reviews` 資料表，儲存每個單字的「下次複習時間」與「熟悉度等級」
3. **間隔重複排程**：最小化版 SM-2：
   - 「認識」→ 下次複習間隔 × 2（上限 30 天）
   - 「不認識」→ 明天重新複習（間隔重置為 1 天）
4. **閃卡 UI**：翻轉動畫（純 CSS），正面法文 / 背面中文，兩個按鈕（認識 / 不認識）
5. **今日待複習計算**：`next_review_at <= 今天` 的單字列為待複習
6. **複習摘要頁**：一輪結束後顯示統計

## Non-Functional Requirements

- 零 npm 依賴：純 HTML/CSS/JS 前端 + `node:http` + `node:sqlite`
- 沿用現有 SQLite DB 檔案（`db/` 目錄下）
- 資料持久化存在本機（SQLite 檔案），不需伺服器
- API 回應 < 100ms（本機 SQLite，單人使用）
- 閃卡翻轉動畫流暢（純 CSS transition）

## Success Criteria

1. 使用者完成 20 張閃卡複習，「認識」比例可在畫面上看到（可量化）
2. 連續複習 3 天後，標記「認識」的單字不會在隔天出現（間隔重複生效）
3. 標記「不認識」的單字在明天仍會出現（重置機制生效）
4. 功能從 0 到可用，不需安裝任何新套件

## Constraints & Assumptions

- 單人本機工具，無帳號系統，無多使用者隔離
- 單字來源為現有 DB，不另建獨立詞庫
- 間隔重複採用最小化 SM-2（不實作完整演算法，避免過度工程）
- 開發語言：JavaScript（Node.js 22+，ESM，`node:sqlite`）
- 前端：原生 HTML/CSS/JS，無框架

## Out of Scope

- 帳號系統 / 多使用者
- 多語言介面（只有中文 UI）
- 手機 App / PWA
- 完整 SM-2 演算法（只做簡化版）
- 語音播放（閃卡只顯示文字）
- 自訂新增單字（使用現有詞庫，不做手動輸入）

## Dependencies

- 現有 `api/db.js`（SQLite 連線）
- 現有 `api/words.js`（單字資料）
- 現有前端結構（`public/`）
- Node.js 22+（`node:sqlite` 內建模組）
