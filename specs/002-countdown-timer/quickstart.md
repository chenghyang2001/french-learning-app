# 快速驗證指南：倒數計時器

**功能**：`002-countdown-timer`
**合約**：[API 合約](contracts/api.md) | **資料模型**：[data-model.md](data-model.md)

---

## 前置條件

- Node.js **22.5.0 以上**（需要內建 `node:sqlite` 模組）
- 確認版本：`node --version`
- **無需** `npm install`（零外部依賴）

---

## 啟動應用程式

```bash
# 切換到功能根目錄
cd countdown-timer/

# 啟動伺服器（首次執行會自動建立 countdown.db）
node server.js
```

預期輸出：

```
伺服器啟動於 http://localhost:3000
資料庫已就緒：countdown.db
```

---

## 驗證情境

### 情境 1：首次開啟，空白頁面

1. 瀏覽器開啟 `http://localhost:3000`
2. **預期**：看到新增活動的表單（名稱欄位 + 日期時間選擇器 + 新增按鈕）
3. **預期**：活動清單區域為空，顯示提示文字（例如「尚無活動，請新增你的第一個活動」）

---

### 情境 2：新增一個未來的活動並看到倒數

1. 在名稱欄位輸入「生日派對」
2. 選擇一個明天同一時間的日期（例如 2026-06-29T10:00）
3. 點擊「新增」按鈕
4. **預期**：表單清空，活動清單出現「生日派對」卡片
5. **預期**：卡片顯示「0 天 23 小時 XX 分鐘」（XX 為實際剩餘分鐘，誤差 ≤ 1 分鐘）

---

### 情境 3：倒數自動更新

1. 延續情境 2 的頁面（不重新整理）
2. 等待約 1 分鐘
3. **預期**：倒數分鐘數自動減 1，無需手動重新整理

---

### 情境 4：新增已過期的活動

1. 輸入活動名稱「去年的事」
2. 選擇昨天的日期時間
3. 點擊「新增」
4. **預期**：卡片顯示「已到期」或類似標記，而非負數倒數

---

### 情境 5：刪除活動

1. 延續前面情境，頁面上有至少一個活動
2. 點擊任一活動卡片上的刪除按鈕
3. **預期**：確認提示（或直接刪除，視實作而定）
4. **預期**：該卡片從清單消失，其餘活動不受影響

---

### 情境 6：頁面重整後資料仍保留（SQLite 持久化）

1. 確認頁面上有至少一個活動
2. 在瀏覽器中按 F5 重新整理
3. **預期**：活動仍然存在，且倒數更新為最新時間

---

### 情境 7：表單驗證

1. 不輸入名稱，直接點擊「新增」
2. **預期**：無法送出，頁面顯示錯誤提示「名稱不可為空白」
3. 不選擇日期，直接點擊「新增」
4. **預期**：無法送出，頁面顯示錯誤提示

---

## API 快速驗證（curl）

```bash
# 取得所有活動
curl http://localhost:3000/api/activities

# 新增活動
curl -X POST http://localhost:3000/api/activities \
  -H "Content-Type: application/json" \
  -d '{"name":"生日派對","target_at":"2026-12-31T23:59"}'

# 刪除活動（將 1 替換為實際 ID）
curl -X DELETE http://localhost:3000/api/activities/1
```
